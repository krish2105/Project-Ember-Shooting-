# Decisions

| ID | Decision | Rationale |
|---|---|---|
| DEC-001 | Remain third-person at all times | Non-negotiable product definition |
| DEC-002 | Implement one 1 km² harbor slice; document the 4 × 4 km world | Validates the loop before world expansion |
| DEC-003 | Use C++ runtime modules and designer-facing Primary Data Assets | Reliability with tunable content |
| DEC-004 | Do not adopt Gameplay Ability System for the first slice | Current combat/status scope does not justify its dependency cost |
| DEC-005 | Use bounded Recast navigation for critical encounters | World Partition navigation must not be a shipping dependency until validated |
| DEC-006 | State-machine animation is mandatory fallback | Motion matching depends on asset coverage and measured Mac cost |
| DEC-007 | Hardware RT and MegaLights stay disabled | Unsupported on the target Mac path |
| DEC-008 | Final public artifact is arm64, ad-hoc-signed and unnotarized | No Developer ID identity is available |
| DEC-009 | Functional placeholders are allowed only when tracked | Prevents missing assets from being misrepresented as final content |
